# 关注池自动化规则

本项目的关注池自动化遵循防守优先：先剔除低质量机会，再补充新候选；先确认数据新鲜，再发布到 GitHub。

## 自动运行时间

- 运行时间：中国时间每周一至周五 18:00。
- GitHub Actions cron：`0 10 * * 1-5`。
- 发布闸门：只有 RSSCAST 最新市场快照日期等于当天中国日期，才导出并提交公开关注池。
- 节假日处理：如果交易所休市导致快照不是当天，工作流会运行但跳过发布。

## 旧票保留与剔除

旧票先经过 `scripts/refresh_before_analysis.py`，该入口会同步行情并调用 `scripts/refresh_watchlist.py`。

状态分层：

- `重点关注`：成交额、波动或涨跌幅仍能说明资金活跃，短线强度分数高。
- `观察`：仍有资金敏感度，但强度不足以排在最前。
- `低频观察`：缩量、低流动性、状态过旧或资金敏感度下降。
- `剔除候选`：跌破失效位、放量冲高回落、连续缩量窄幅或强度明确失效。

硬剔除规则：

- 自动化运行时使用 `--prune-removal-days 1`。
- 非持仓股第一次失效先标记为 `剔除候选`。
- 如果下一次有效刷新仍是 `剔除候选`，则将 `stocks.is_watchlisted` 置为 `0`，不再进入公开关注池。
- 持仓股不参与硬剔除，只保留在风险管理范围内。

## 新票加入规则

新票来源不是全市场无边界扫描，而是 `config/watchlist_candidates.json` 里的显式候选宇宙。这样能控制能力圈，避免把注意力浪费在随机噪音上。

候选票必须同时满足：

- 不在持仓中。
- 不在当前关注池中。
- 非 ST / 风险警示标的。
- 最新成交额不低于 `20` 亿元。
- 5 日均成交额不低于 `15` 亿元。
- 振幅不低于 `3%`，或绝对涨跌幅不低于 `2%`。
- 当日不能收跌超过 `2%`。
- 不能出现放量长上影、收盘位置过低的冲高回落形态。
- 综合评分不低于 `65`。

评分结构：

- 成交额：最高 `30` 分。
- 相对 5 日均额放量：最高 `25` 分。
- 当日振幅：最高 `15` 分。
- 当日涨幅：最高 `20` 分。
- 接近或突破近 10 日高点：`15` 分。
- 收盘位置靠近日内高位：最高 `10` 分；收盘位置偏低则扣分。

加入上限：

- 单次最多新增 `5` 只。
- 活跃关注池最多 `18` 只。
- 评分 `80` 及以上标记为 `重点关注`，否则标记为 `观察`。

## 发布到 GitHub

工作流完成后只提交公开文件：

- `db/public_seed.sql`
- `docs/data/watchlist_latest.json`
- `docs/watchlist.md`

私有数据库、持仓成本、仓位数量、可用数量和私人复盘不会提交。

数据来源：RSSCAST，https://app-cn.rsscast.io
